# Naming Convention

## Base Rule

- Use short, descriptive English names for folders.
- Use Korean only when the folder is for human-facing notes and the team agrees on it.
- Keep machine-readable names stable across devices.

## Recommended Root Names

```text
iCloud/
  AI-Workspace/

NAS/
  AI-Workspace-backup/

GitHub private repo/
  bss-workspace/
```

## Folder Name Rules

- Use lowercase for technical folders.
- Use hyphens for multi-word names.
- Avoid spaces in folder names.
- Avoid long names that repeat the same meaning.

## File Name Rules

- Use `YYYY-MM-DD` for date prefixes.
- Use `topic-type-version` when the file has a clear role.
- Use `tmp-` for disposable files only.
- Use `final` only for the last approved output.

## Examples

```text
2026-03-25-ai-workspace-rules.md
business-map-reference-v2.pdf
tmp-notes.txt
final-business-architecture.png
```
